# 🎓 College OAIP Engineering Portfolio: Desktop & Core Applications

Comprehensive software repository engineered as the final framework for the **"Fundamentals of Algorithmization and Programming"** course at MTKP. Developed with advanced modular architecture, dynamic database integration, and local data sandboxes.

## ⚙️ Cross-Platform Execution Architecture
The applications feature an isolated runtime configuration depending on the host deployment environment:
* **Production Environment (Standard User GUI):** Executes through the **`main_home`** pipeline for native PyQt5 rendering and database operations.
* **Development & Debugging Environment (Laptop Testing):** Executes through the **`emulator_db`** core layer to bypass hardware limits for smooth local query validation.

---

## 📁 Repository Modules & Component Specification

### 🚨 1. Qt_Imbegging (Core Relational Database Application)
* **Type:** Full-Stack Desktop Graphical User Interface (GUI) Application.
* **Stack:** Python 3.10+ / PyQt5 / SQLite Engine.
* **Core Logic:** A heavy database-driven application handling secure asynchronous data entry, structured record insertion, and high-performance cross-table search parsing. Implements dynamic relational generation of electronic product cards with real-time grid formatting.

### 🎴 2. cards_app (Isolated Identity & Card Generation Module)
* **Type:** Standalone Micro-GUI Application.
* **Stack:** Python / Object-Oriented Layouts.
* **Core Logic:** Dedicated software engine designed specifically for procedural layout rendering, metadata management, and structural formatting of independent entity cards.

### 🕹️ 3. prakt_tkinter_safina (Tkinter UI Sandbox)
* **Type:** Interactive Frontend Sandbox / Prototyping Playground.
* **Stack:** Python / Tkinter GUI Standard Library.
* **Core Logic:** A lightweight interactive script engineered as a development playground to demonstrate master-level comprehension of event loops, pixel-perfect window geometry, widgets manipulation, and standard Tkinter library workflows.

### 🧱 4. strtructure_programming_what_I_can (Core Algorithms Archive)
* **Type:** Consolidated Low-Level Architecture Repository.
* **Stack:** Pure Procedural & Structural Python.
* **Core Logic:** Comprehensive software vault containing core mathematical algorithms, data streams validation, structural loops, and optimal procedural logic models implemented during the academic lifecycle.

---

## 🚀 Key Software Advantages
1. **Decoupled OOP Design:** Strict separation of XML-based layouts (`.ui`) from execution code.
2. **Relational Database Engine:** Fast local transaction logging via custom database schemas.
3. **Dual-Environment Portability:** Flexible execution layout switching between `main_home` and `emulator_db` cores.
